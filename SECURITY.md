# Security

Garmon World is a public-facing repository. It should not receive or publish private project material.

## Do Not Share Private Material

Please do not open issues or pull requests containing:

- private files;
- raw logs;
- local machine paths;
- credentials;
- tokens;
- cookies;
- private headers;
- unpublished working notes;
- exact private implementation details;
- internal report filenames;
- protected hashes;
- private owner paths;
- private schema details;
- hidden channels.

## Reporting A Public-Safety Concern

If you notice that a public document seems too detailed, too strong, or unclear about boundaries, please open a careful issue using the review feedback template.

Useful reports include:

- unclear public wording;
- a claim that sounds too strong;
- static/shadow evidence framed as live behavior;
- Body Shadow framed as command authority;
- body contours framed as memory, truth, behavior, or final speech;
- a document that may need a safer summary;
- a missing boundary note.

## Not A Security Bug

This repository does not currently contain a production service, public API, or runnable public system.

Most concerns at this stage are public-safety and documentation-boundary concerns.

Examples of public-safety concerns:

- wording that implies an interactive public demo exists;
- wording that implies body-contour distinction is live behavior;
- wording that implies memory or residue is active;
- wording that implies public summaries reveal the private mechanism.

## Response Style

Reports will be handled slowly and carefully.

The first response may be to clarify the boundary, not to publish more detail.

The goal is to keep the public surface useful without exposing private implementation details.
