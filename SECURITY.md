# Security

Garmon World is a public-facing repository. It must not receive or publish private project material, private implementation details, raw logs, credentials, or internal runtime information.

This repository is meant to describe Garmon at a public-safe level: ideas, boundaries, current public status, and reviewable summaries only.

## Do Not Share Private Material

Please do not open issues, pull requests, comments, or attachments containing:

- private files;
- raw logs;
- local machine paths;
- credentials;
- tokens;
- cookies;
- private headers;
- environment variables;
- unpublished working notes;
- exact private implementation details;
- internal report filenames;
- protected hashes;
- private owner paths;
- private schema details;
- hidden channels;
- private runtime traces;
- internal configuration details.

If a concern requires private context, do not paste the private material into a public GitHub issue. Describe the concern at a high level instead.

## Reporting a Public-Safety Concern

If you notice that a public document seems too detailed, too strong, or unclear about boundaries, please open a careful issue using the review feedback template.

Useful reports include:

- unclear public wording;
- a claim that sounds too strong;
- static/shadow evidence framed as live behavior;
- Body Shadow framed as command authority;
- body contours framed as memory, truth, behavior, or final speech;
- memory or residue framed as active live authority;
- pre-selected boundary language framed as an actual selected result;
- a document that may need a safer summary;
- a missing boundary note;
- wording that could confuse public evidence with private runtime behavior.

## Not a Security Bug

This repository does not currently contain a production service, public API, runnable public system, or interactive public demo.

Most concerns at this stage are public-safety and documentation-boundary concerns, not conventional software security bugs.

Examples of public-safety concerns:

- wording that implies an interactive public demo exists;
- wording that implies body-contour distinction is live behavior;
- wording that implies memory or residue is active authority;
- wording that implies static/shadow checks prove consciousness, life, or mature autonomy;
- wording that implies public summaries reveal the private mechanism;
- wording that implies Garmon has public live choice or actual selected results.

## What This Repository Should Keep Clear

Public Garmon materials should preserve these boundaries:

- internal state is not command authority;
- meaning candidates are not behavior;
- memory or residue is not a hidden motor;
- speech is not the source of choice;
- evidence is not decision truth;
- visibility is not authority;
- static/shadow evidence is not live runtime proof;
- public summaries are not private implementation disclosure.

## Response Style

Reports will be handled slowly and carefully.

The first response may be to clarify the boundary, not to publish more detail.

The goal is to keep the public surface useful, reviewable, and honest without exposing private implementation details or overstating what the current public evidence shows.
