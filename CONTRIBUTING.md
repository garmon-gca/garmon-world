# Contributing

Garmon World is in an early public orientation stage.

The best contribution right now is calm technical feedback.

## Useful Contributions

Helpful contributions include:

- clearer wording;
- safer public framing;
- questions for reviewers;
- suggestions for a minimal demo;
- evaluation ideas;
- references to relevant public work.

## Before Opening An Issue

Please read:

- `README.md`
- `docs/PUBLIC_BOUNDARIES.md`
- `docs/FIRST_PUBLIC_DEMO_PLAN.md`
- `docs/FOR_REVIEWERS.md`

## Good Issue Style

Good issues are:

- specific;
- calm;
- public-safe;
- focused on one point;
- written as feedback or a question, not pressure.

Example:

> The demo plan would be easier to review if it included a small comparison table.

## Please Avoid

Please avoid:

- asking for private files;
- asking for raw logs;
- asking for local paths;
- asking for unpublished working notes;
- asking for exact private implementation details;
- framing the project as a finished product;
- pushing for strong claims before a reproducible demo exists.

## Pull Requests

Pull requests may be useful later.

For now, small documentation suggestions are more useful than code changes.

If you propose a change, please keep it modest and public-safe.

## Tone

The desired tone is:

- careful;
- technical;
- low-pressure;
- honest about limits.

The goal is not hype. The goal is better review.
