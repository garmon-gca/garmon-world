# One-Minute Example

Note: this is an explanation example, not an interactive demo.

This page gives a short public-safe example of the Garmon idea:

> an answer may have an origin before speech.

The goal is not to show a finished system.

The goal is to make the pre-speech path easier to understand.

## The Problem

An LLM can write useful text.

It can answer clearly, politely, and intelligently.

But the final text does not always show what happened before the text was written.

Several things can stay hidden:

- what the event meant before speech;
- whether different body-contour states stayed distinct before speech;
- whether the final answer expressed a prior response direction or only generated text directly;
- whether speech expressed a path or invented the path after the fact.

Garmon studies this pre-speech layer.

A compact public frame is:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

## Example 1: Event Reading Before Speech

Event:

> A tense message arrives, and the user wants to answer without making the situation worse.

A direct text-task shape might look like this:

```text
request -> generate polite reply
```

This can produce useful text.

But it may hide the step where the event receives meaning.

A Garmon-shaped public example adds a visible pre-speech reading:

```text
lower intensity
keep contact without pressure
avoid escalation
```

From that reading, a response direction can form:

```text
calm boundary reply
```

Only after that does the LLM speech layer write the final answer.

Public chain:

```text
event -> public event reading -> response direction -> LLM speech -> final answer
```

The point is not that the final answer becomes more dramatic.

The point is that the path before speech becomes easier to inspect.

## Example 2: Body-Contour Distinction

Different body-contour candidates can shape the pre-speech reading differently.

Public-safe examples:

```text
quiet baseline -> stable low-pressure contour
fatigue -> low-energy warning contour
recovery -> downshift / return contour
stress -> tension after event contour
mixed stress/recovery -> tension plus return contour
night/downshift -> body-time downshift contour
day/wake support -> body-time wake-support contour
```

A simpler system might flatten all of these into one vague label:

```text
state changed
```

That would hide the difference between quiet baseline, fatigue, recovery, stress, mixed stress/recovery, and rhythm.

Body Shadow is useful because it makes this distinction reviewable before final speech.

The achievement is not that the contours become commands.

The achievement is that the contours remain distinguishable.

## Example 3: Boundary

A body contour can help shape a public event reading.

But it must not become command authority.

Safe public chain:

```text
body contour -> meaning candidate -> passive context
```

This is a review surface.

It is not:

```text
body contour -> command
body contour -> memory
body contour -> behavior
body contour -> truth
body contour -> final speech
body contour -> selected result
```

The boundary matters because Garmon studies the path before speech without turning every visible trace into authority.

Short law:

```text
participation is not power
```

A contour may participate in the path.

It must not own the answer.

## Example 4: Speech Layer

The LLM remains important.

It writes the final words.

But in Garmon, speech should express a prior direction rather than create the whole path by itself.

Safe public shape:

```text
response direction -> LLM speech -> final answer
```

Unsafe interpretation:

```text
LLM speech -> choice
LLM speech -> origin
LLM speech -> truth
```

The LLM is the speech layer.

It is not presented as the whole system.

## What This Example Shows

This example shows a public-safe idea:

- an event can be read before final wording;
- body-contour candidates can remain distinguishable before speech;
- a response direction can be described before final answer text;
- boundaries can prevent contours from becoming command, memory, behavior, truth, or selected result;
- speech can be framed as expression rather than source.

## What This Example Does Not Show

This example does not show:

- a finished product;
- an interactive public demo;
- live behavior;
- live choice;
- an actual selected result;
- a new trained language model;
- subjective experience;
- consciousness;
- biological life;
- mature autonomy;
- general intelligence;
- external validation;
- memory influence on choice;
- behavior authority from body contours;
- speech as the source of choice;
- publication of the private mechanism.

These limits do not make the example empty.

They keep the example accurate.

## Why This Matters

The useful question is not only whether the LLM can write nice text.

The useful question is:

> can the pre-speech layer be made easier to inspect?

Garmon studies whether internal-state-shaped event reading can be shown before final speech while preserving boundaries between state, meaning, memory, choice, behavior, truth, evidence, and speech.

## One-Line Summary

A final answer can be useful, but Garmon asks what happened before it became words.

This example shows the public idea:

```text
event -> body-contour-shaped reading -> response direction -> LLM speech
```

The path becomes visible.

The boundary remains closed.
