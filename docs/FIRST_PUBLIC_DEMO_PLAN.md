# First Public Demo Plan

This document describes the intended first public artifact path for Garmon.

The first public step does not need to be an interactive chat demo.

A better first step is likely a small, bounded, public-safe artifact that makes the current Body Shadow achievement easier to inspect.

The goal is not to prove a large claim.

The goal is to create a clear, reviewable, reproducible starting point for technical feedback.

## Core Public Question

The first artifact should help answer one narrow question:

> can body-contour candidates remain distinguishable before final LLM speech?

A later artifact may help explore a second question:

> can a public event-reading step become visible before final speech?

A broader Garmon direction sits behind these questions:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This broader direction is not a public claim of live choice, live behavior, consciousness, life, or mature autonomy.

The artifact should help reviewers inspect a small part of the path before speech.

## Current Best First Artifact

The strongest current first artifact direction is a body-contour distinction artifact.

The artifact should show that several body-contour candidates remain distinguishable before speech.

Public-safe contour examples include:

- quiet baseline;
- fatigue;
- recovery;
- stress;
- mixed stress/recovery;
- night/downshift;
- day/wake support.

The important point is not stronger wording.

The important point is a more inspectable pre-speech layer.

The artifact should help reviewers see whether body-contour candidates remain distinct or collapse into one generic label.

## What The First Artifact Should Show

The first body-contour artifact should show:

- a small set of body-contour candidates;
- one public-safe reading for each contour;
- a small comparison section;
- a visible boundary section;
- a short limits section;
- a short “what this does not prove” section.

A useful minimal shape may look like:

```text
same event
-> several body contours
-> different public-safe readings
-> different response directions
```

The artifact should make the distinction easier to inspect.

It should not pretend to be a full public system.

## Example Artifact Shape

A small comparison may look like this:

```text
quiet baseline
-> stable low-pressure reading
-> calm response direction

fatigue
-> low-energy caution reading
-> reduced-pressure response direction

stress
-> tension contour reading
-> escalation-avoidance direction
```

The point is not dramatic wording.

The point is whether the contours remain distinguishable before final speech.

## Later Event-Reading Artifact

A later artifact may focus more directly on event meaning before speech.

A small future example could include:

- one short visible event;
- two or more close response directions;
- a visible event-reading step;
- a final answer following the response direction used by the artifact;
- a comparison note;
- a limits section.

A possible public-safe comparison:

```text
direct text task
vs
event -> reading -> response direction -> speech
```

The later artifact should still remain small and bounded.

A tiny reproducible check is better than a large unclear demo.

## What The Artifact Should Avoid

The artifact should avoid:

- private implementation details;
- raw internal logs;
- local machine paths;
- protected internal identifiers;
- unpublished working notes;
- private runtime material;
- private diagnostic material;
- private configuration material;
- hidden internal structure;
- claims stronger than the evidence;
- dramatic presentation that hides the boundary.

The artifact should explain the principle.

It should not expose the protected mechanism.

## Reviewable Output

The output should be readable without knowledge of the private project.

An outside reviewer should be able to see:

- which body-contour candidates are being compared;
- what public-safe reading is attached to each contour;
- whether the contours stay distinct;
- what the result may suggest;
- what the result does not prove.

For a later event-reading artifact, an outside reviewer should be able to see:

- what the event was;
- what public event reading appeared before final speech;
- what response direction was used;
- what final answer followed that direction;
- what changed between close response directions;
- what the result may suggest;
- what the result does not prove.

The artifact should be understandable without requiring trust in hidden internals.

## Minimal Comparison Goal

A useful first comparison may:

- show quiet baseline, fatigue, recovery, stress, mixed stress/recovery, and rhythm as separate body-contour candidates;
- give each one a small public-safe reading;
- compare whether they remain distinct or collapse into one generic label.

A useful later comparison may:

- show the same event as a direct text task;
- show the same event through a visible event-reading step;
- compare whether the final answer follows a clearer response direction.

Both comparisons should stay small.

The first artifact should be inspectable in minutes, not hours.

## Success Criteria

The first public artifact is useful if:

- it is easy to inspect;
- it is understandable without private internals;
- the body-contour distinction is visible;
- the artifact avoids strong claims;
- the boundary remains visible;
- the artifact creates a clear next technical question;
- the artifact can be repeated and improved after review.

The artifact succeeds if reviewers can say:

> I understand what is being compared and what is not being claimed.

## Failure Criteria

The artifact is not ready if:

- it depends on private material;
- it exposes raw logs or private runtime traces;
- it only shows different wording without contour distinction;
- body-contour differences collapse into vague labels;
- it requires trust in hidden steps;
- it sounds like a finished product;
- it implies live behavior;
- it implies actual selected result;
- it implies public runtime execution;
- it cannot be repeated or inspected by an outside reader.

## Boundary

The first public artifact must not imply:

- live behavior;
- live choice;
- actual selected result;
- active memory influence on choice;
- memory authority;
- truth authority;
- speech authority;
- behavior authority;
- command authority;
- implementation permission;
- runtime proof;
- consciousness;
- biological life;
- mature autonomy.

These boundaries do not weaken the artifact.

They keep the artifact accurate.

## Relation To Memory And Trace

The first body-contour artifact should not frame contour distinction as memory.

A body contour is not memory.

A diagnostic trace is not lived residue.

A static/shadow check is not lived experience.

Memory or residue should remain future-gated unless a separate artifact and separate boundary review support a narrower claim.

A safe current chain is:

```text
event -> body contour -> visible distinction
```

A stronger future chain would require separate evidence:

```text
event -> body-level shift -> reviewed trace -> bounded future influence
```

The second chain is not claimed by the current public artifact.

## Relation To Speech

The first body-contour artifact should not make final speech the only visible source of meaning.

The point is to show something before speech.

In a later event-reading artifact, final speech should follow a response direction rather than create the whole meaning after the fact.

Safe public framing:

```text
response direction -> speech
```

Unsafe framing:

```text
speech -> choice
speech -> meaning
```

## Public Review Focus

A reviewer should be able to ask:

- Are the contours clearly distinguishable?
- Is the event-reading step understandable?
- Is the response direction understandable?
- Is the boundary preserved?
- Does the artifact avoid sounding like live behavior?
- Does the artifact avoid sounding empty?
- Is the distinction easier to inspect than final text alone?
- What small improvement would make the artifact clearer?

## Next Action

The next action is to design the smallest useful public-safe body-contour artifact:

- contour set;
- public-safe readings;
- comparison section;
- boundary section;
- limits section;
- short “does not prove” section.

Only after that should runnable code, interactive demo material, or event-reading demo code be considered.

The first useful public step should be:

- small;
- bounded;
- inspectable;
- technically honest;
- easy to review.

## One-Line Summary

The first public artifact should be a small, bounded body-contour distinction artifact: it should show inspectable differences before final LLM speech without implying live behavior, memory authority, actual selected results, runtime proof, consciousness, life, or a finished system.
