# Public Glossary

This glossary explains public terms used in Garmon World.

It avoids private implementation details.

The goal is to make the public surface easier to read and review without exposing the protected mechanism.

## Answer origin before speech

The central public idea of Garmon.

It means that an artificial answer can be studied not only as final generated text, but as something that may have a path before it becomes words.

A compact public frame is:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This is a research frame, not a claim that the full private system is publicly open.

## Internal state

The public term for the inner background from which Garmon reads an event.

Internal state may shape how an event is interpreted before final speech.

It is not command authority.

It is not a public claim of subjective feeling, biological life, or consciousness.

## Bodily background

A public phrase for the internal background that gives an event weight before speech.

This does not mean a published biological simulation or a human-like body.

It means that the event is not treated as plain text only.

## Body contour

A public-safe shape of internal state around an event.

Examples include:

- quiet baseline;
- fatigue;
- recovery;
- stress;
- mixed stress/recovery;
- day/night rhythm.

A body contour is not:

- a command;
- memory;
- behavior;
- truth;
- final speech;
- chooser authority;
- selected-result authority;
- implementation permission.

## Body-born meaning

The meaning an event may receive through an internal-state layer before final LLM speech.

In Garmon World, this is a central public term.

It means that a request is not only text to process. It can be treated as an event that receives meaning before the LLM writes final words.

Body-born meaning is not:

- a biological claim;
- a claim of subjective experience;
- a claim of consciousness;
- a claim of mature autonomy;
- a claim that Garmon copies the human body.

## Body Shadow

A public term for static/shadow checks that examine whether body-contour candidates remain distinct before speech.

Body Shadow is a review surface.

It is not:

- live interaction;
- live behavior;
- subjective experience;
- runtime proof;
- command authority;
- memory authority;
- selected-result authority.

Body Shadow gives Garmon a public-safe way to show body-contour distinction before final LLM speech.

## Pre-speech layer

The public term for the layer before the LLM writes final text.

In Garmon World, the pre-speech layer may include public-safe descriptions of:

- internal state;
- body contour;
- event reading;
- meaning candidate;
- response direction;
- boundaries.

It is not a public claim of live choice, live behavior, or public runtime operation.

## Event

A public term for an incoming situation, request, signal, or message treated as something the system reads before speech.

An event is not only a text task.

In Garmon’s public language, an event can meet an internal state and receive a body contour before final wording.

## Event reading

The public term for how an event is interpreted before final speech.

An event reading may shape response direction.

It is not:

- a command;
- final behavior;
- decision truth;
- the private mechanism.

## Meaning candidate

A public-safe candidate for what an event may mean before final speech.

The word “candidate” is important.

It means the meaning is not yet selected result, behavior, truth, or command.

A meaning candidate may participate in the path.

It must not take power.

## Response direction

The public name for the direction that final speech should follow.

Example:

```text
lower intensity, keep contact without pressure
```

A response direction is not the same as final speech.

It is also not a public claim of live choice or actual selected result.

## LLM speech layer

The part of the public frame where an LLM writes final text.

The LLM speech layer is important.

It makes the answer readable to a human.

But in Garmon World, the LLM speech layer is not presented as:

- the whole system;
- the source of choice;
- the origin of meaning;
- proof of autonomy.

Short form:

```text
LLM = speech layer
```

## Speech

The visible language output.

Speech expresses a path.

Speech should not replace the path.

In public Garmon terms, speech is not:

- the source of choice;
- decision truth;
- memory;
- proof of consciousness;
- proof of life;
- proof of mature autonomy.

## Trace

A public term for the mark a past event may leave.

Trace can mean changed sensitivity, caution, recovery pressure, or another bounded after-effect.

Trace is not command.

Trace is not a hidden motor.

Trace is not automatically memory authority.

## Memory as trace

A public phrase for the idea that memory-like material should be treated as trace, not as a hidden motor.

A safe public chain is:

```text
past event -> trace -> changed sensitivity -> present meaning
```

An unsafe interpretation would be:

```text
past event -> hidden command -> current result
```

This repository does not claim active body memory, memory influence on live choice, or lived biography.

## Residue

A possible future trace left by an event after recovery and review.

Residue-like claims require separate evidence, provenance, and boundary review.

A diagnostic after-effect, warning, static check, or public artifact is not automatically residue.

In public materials, residue should be treated as future-gated evidence.

## Line of behavior

A public phrase for a continuing direction of behavior over time, rather than a single isolated answer.

In this repository, it is used cautiously.

It does not mean proven consciousness, personhood, biological life, or mature autonomy.

## Static check

A non-runtime check that can show distinction without opening live interaction.

A static check can help review whether a boundary is preserved.

A static check is not runtime proof.

## Shadow check

A bounded review check that inspects a public-safe shadow surface without publishing or activating the private mechanism.

A shadow check is not live behavior.

## Static/shadow evidence

Public-safe evidence from static or shadow checks.

It may support reviewability and boundary clarity.

It does not prove:

- consciousness;
- biological life;
- subjectivity;
- mature autonomy;
- live behavior;
- runtime execution;
- the whole theory.

## Negative control

A deliberately invalid case used to check that forbidden authority leaks are caught.

A useful check should pass valid cases and fail when a forbidden authority leak is introduced.

Example:

```text
valid contour summary -> should pass
contour framed as command -> should fail
```

Negative controls help keep the public claim honest.

## Passive context

A public term for context that can be visible for review without becoming chooser input or behavior authority.

Passive context is not influence by itself.

Passive context is not live choice.

## Visibility

The fact that something can be seen, inspected, or summarized.

Visibility is not authority.

A visible layer does not automatically become a layer with power.

## Influence

A possible effect on interpretation or context.

Influence is not automatically authority.

Influence is not automatically choice.

## Authority

The right or power of a layer to affect decision, behavior, memory, or implementation.

Authority must not be implied from visibility alone.

The public law is:

```text
participation is not power
```

## Choice boundary

A public term for the protected place where one path may become result.

In this repository, choice boundary language is used carefully.

It does not mean that public live choice has been opened.

## Choice visibility

A public term for making a choice-related boundary visible for review.

Choice visibility is not live choice.

It is a review concept, not a claim of public selected result.

## Choice ownership boundary

A public term for the boundary that prevents ownership of choice from being transferred to context, speech, reports, or memory.

It is not a claim that public live choice has been opened.

## Behavior boundary

A public term for the boundary before behavior can be claimed.

Context alone does not create behavior.

A behavior boundary is not public live behavior.

## Static choice boundary chain

The public-safe sequence of pre-speech layers from body contour through meaning candidate, passive context, choice visibility, choice ownership, and behavior boundary.

A public form is:

```text
body contour
-> meaning candidate
-> passive context
-> choice visibility
-> choice ownership
-> behavior boundary
```

This is static/shadow evidence.

It verifies that body-contour candidates can move along this path without becoming commands, memory, behavior, truth, speech authority, selected-result authority, runtime proof, or implementation permission.

It is not public live choice.

## Pre-selected result boundary candidate

A public term for the synthetic marker just before an actual selected result.

It indicates that the door next to selection has been named.

It does not mean the door has been opened.

Correct public meaning:

```text
boundary named
boundary not opened
```

No actual selected result, live choice, behavior, application authority, memory influence, or runtime proof is claimed.

This boundary is evidence only and does not grant implementation permission.

## Public-facing container

A public-facing container is an outward form for explanation, summary, or representation.

It is not:

- the organism;
- memory;
- truth;
- the chooser;
- life-status evidence;
- consciousness evidence.

Public-facing containers should explain the public surface without exposing the private mechanism.

## Controlled implementation gate

A public boundary principle.

It means that review evidence is not implementation permission.

A successful check does not automatically open a higher layer.

Any future implementation step requires separate approval, narrow scope, and boundary review.

## Evidence

Material that supports a specific public claim or boundary.

Evidence is not decision truth.

Evidence is not permission.

Evidence is not proof of the whole theory.

A good evidence statement says what it supports and what it does not support.

## Decision truth

The source that would count as actual decision truth inside the private system.

Public reports, summaries, artifacts, and evidence notes are not decision truth.

This distinction matters because a public explanation should not become the decision it describes.

## Report

A public or private description of what was reviewed, checked, or observed.

A report can support evidence.

A report is not decision truth.

A report is not permission.

## Public artifact

A small reviewable output that can be inspected without private internals.

The first useful artifact does not have to be an interactive demo.

A public artifact should remain:

- small;
- bounded;
- reviewable;
- public-safe;
- clear about what it shows;
- clear about what it does not show.

## Public-safe

Safe for public description without exposing private project material, private implementation details, raw internal logs, local paths, protected internal identifiers, private runtime material, or protected mechanism details.

Public-safe does not mean complete.

It means suitable for public review.

## Private mechanism

The protected internal working structure that is not published in this repository.

Public materials may explain the principle and public review surface.

They must not expose or reconstruct the private mechanism.

## Runtime proof

Evidence that a live runtime system performed a behavior.

Static/shadow evidence is not runtime proof.

This repository does not currently claim public runtime proof.

## Public research preview

The current public status of Garmon World.

It means the repository is open for reading, orientation, and careful review.

It does not mean product launch, public live system, or finished demo.

## One-line summary

Garmon World uses these terms to describe a bounded public review surface: body contours and event readings can be inspected before final LLM speech without becoming command, memory, behavior, truth, speech authority, selected-result authority, runtime proof, or implementation permission.
