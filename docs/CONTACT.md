# Contact

Garmon is currently in an early public research stage.

The best contact style is calm technical feedback: specific, low-pressure, and focused on the public review surface.

This repository is not the right place to request private internals, raw logs, protected artifacts, or implementation details.

## Good Reasons To Reach Out

You can reach out if you want to discuss:

- whether the Body Shadow / body-contour distinction framing is understandable;
- what a small body-contour artifact should show;
- whether the static choice chain is understandable as bounded public-safe evidence;
- whether the pre-selected result boundary candidate is clearly not an actual selected result;
- how to make a small demo or artifact more reproducible;
- how to evaluate the event-reading step before LLM speech;
- how to keep public claims clear without hiding the achievement;
- how to explain the boundary between internal state, meaning, memory, choice, behavior, truth, and speech;
- how to keep memory/residue language from sounding like active live authority;
- how to keep LLM speech clearly framed as a speech layer, not the source of choice;
- how to write safer technical explanations;
- relevant papers, projects, or benchmarks.

## Preferred First Message

A useful first message can be simple:

> I read the Garmon public notes. I am interested in the Body Shadow / body-contour distinction question. Here is one suggestion or question.

Another useful form:

> The public framing was clear to me up to this point: [...]. The part I would simplify is [...].

A useful static-chain-focused message can be:

> I understood the static choice chain as a bounded review surface, not as a claim of live choice. The boundary I would clarify is [...].

A useful boundary-focused message can be:

> I understood that memory/residue is not being claimed as active live authority. The wording I would make clearer is [...].

## What To Avoid

Please avoid:

- asking for private internals;
- asking for raw logs;
- asking for unpublished local files;
- asking for private implementation details;
- asking for protected hashes;
- asking for internal report filenames;
- asking for private owner paths;
- asking for private schema details;
- treating static/shadow evidence as live behavior;
- treating the static choice chain as live choice;
- treating the pre-selected result boundary candidate as an actual selected result;
- treating memory or residue as active live authority;
- treating LLM speech as the source of choice;
- framing the project as a finished product;
- framing Garmon as conscious, alive, or maturely autonomous;
- pushing for strong claims before a public artifact exists.
