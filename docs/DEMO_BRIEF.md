# Demo Brief

This document is a public-safe outline for future Garmon demo artifacts.

The next public artifact does not need to be an interactive chat demo.

A better first step is likely a small, bounded, reviewable artifact that shows what has already become visible in the current public-safe layer.

The goal is not to prove a large claim.

The goal is to make one small pre-speech layer easier to inspect.

## Core Goal

The first public artifact should help answer one narrow question:

> can body-contour candidates remain distinguishable before final LLM speech?

A second useful question for later artifacts is:

> can an event receive a visible public reading before final speech?

A broader public Garmon direction sits behind these questions:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This broader direction is not a public claim of live choice, live behavior, consciousness, life, or mature autonomy.

It is a way to keep the path before final speech reviewable without collapsing state, meaning, memory, choice, behavior, truth, and speech into one text-generation step.

## Current Public-Safe Achievement

The clearest current public-safe achievement is Body Shadow.

Static/shadow checks show that several body-contour candidates remain distinguishable before speech.

Public-safe examples include:

- quiet baseline;
- fatigue;
- recovery;
- stress;
- mixed stress/recovery;
- night/downshift;
- day/wake support.

The important result is not stronger wording.

The important result is that these contours do not collapse into one generic label before final text is written.

This gives Garmon a small but meaningful pre-speech review surface.

## Best First Artifact

The strongest first artifact direction is currently a body-contour distinction artifact.

It should remain:

- small;
- reviewable;
- bounded;
- public-safe;
- understandable in minutes.

A useful artifact should show:

- a small contour set;
- one public-safe reading for each contour;
- a visible distinction between contours;
- a clear limits note;
- a clear boundary note.

The artifact should show achievement first.

The boundary should remain visible immediately after.

## Example Artifact Shape

A small public-safe comparison may look like:

```text
quiet baseline -> stable low-pressure contour
fatigue -> low-energy caution contour
recovery -> downshift / return contour
stress -> tension contour
mixed stress/recovery -> recovery under remaining pressure
night/downshift -> body-time downshift contour
day/wake support -> body-time wake-support contour
```

The useful review question is:

> do these contours remain distinguishable before speech?

The useful point is not dramatic language.

The useful point is inspectable distinction.

## Event-Reading Direction

A later event-reading artifact may explore meaning before final speech more directly.

A small future example could include:

- one visible event;
- a public-safe event reading;
- two close response directions;
- one response direction used;
- a final answer following that direction;
- a comparison note;
- a limits section.

Possible public-safe comparison:

```text
direct text response
vs
event -> reading -> response direction -> speech
```

The event-reading layer should remain reviewable.

It should not become command authority.

## What The Artifact Should Show

The artifact should help an outside reviewer see:

- which body-contour candidates are being compared;
- what public-safe reading is attached to each contour;
- whether the contours remain distinct;
- what the result may suggest;
- what the result does not prove.

A later event-reading artifact should also help reviewers see:

- what the event was;
- what reading appeared before speech;
- what response direction was used;
- what changed between close response directions;
- how speech followed the direction.

The artifact should be understandable without access to the private mechanism.

## Boundary Notes

Every artifact should keep a short boundary note close to the result.

For body-contour artifacts:

```text
body contour is not command
body contour is not memory
body contour is not behavior
body contour is not truth
body contour is not final speech
```

For event-reading artifacts:

```text
event reading is not command
response direction is not selected result
speech is not proof of live choice
```

For both:

```text
static/shadow evidence is not runtime proof
public summary is not the private mechanism
visibility is not authority
evidence is not decision truth
```

The artifact should preserve the public law:

```text
participation is not power
```

## What The Artifact Must Avoid

The artifact must avoid:

- private project material;
- raw internal logs;
- local machine paths;
- protected internal identifiers;
- unpublished working notes;
- private runtime material;
- private diagnostic material;
- private configuration material;
- any description that reconstructs the protected mechanism;
- claims stronger than the evidence.

If a detail is useful only because it makes the project sound stronger, it should stay out.

## Expected Public Output

The output should be small and readable.

For a body-contour artifact, it may include:

- a contour list;
- public-safe readings;
- a small comparison note;
- a short boundary section;
- a short “does not show” section.

For a later event-reading artifact, it may include:

- one short event;
- one public-safe event reading;
- one response direction;
- one final answer;
- one comparison note;
- one limits section.

The output should not require hidden context.

## Success Criteria

The first public artifact should be considered useful if:

- an outside reader can inspect it quickly;
- the output is small enough to review;
- the body-contour distinction is understandable;
- the event-reading step is understandable if included;
- the claims remain modest;
- the private internals remain private;
- the boundary stays visible;
- the result creates a clear next technical question.

A useful artifact should leave the reviewer thinking:

> I understand what is being shown and what is not being claimed.

## Failure Criteria

The artifact is not ready if:

- it depends on private material;
- it exposes raw logs or private runtime traces;
- it requires trust in hidden steps;
- it only shows different wording;
- body-contour differences collapse into vague labels;
- the event-reading step is unclear;
- it sounds like a finished product;
- it implies live behavior;
- it implies actual selected results;
- it implies memory influence on live choice;
- it implies runtime proof;
- it cannot be repeated or inspected by an outside reader.

## Boundary

The artifact must not imply:

- live behavior;
- live choice;
- actual selected result;
- active memory influence on choice;
- memory authority;
- truth authority;
- speech authority;
- behavior authority;
- command authority;
- implementation permission;
- runtime proof;
- consciousness;
- biological life;
- mature autonomy.

These limits do not weaken the artifact.

They keep the artifact accurate.

## Relation To Memory

The first body-contour artifact should not frame contour distinction as memory.

A body contour is not memory.

A diagnostic trace is not lived residue.

A static/shadow check is not lived experience.

Memory and residue should remain future-gated unless separate evidence and separate boundary review support a narrower claim.

## Relation To Speech

The first artifact should not make final speech the only visible source of meaning.

The point is to show something before speech.

In a later event-reading artifact, final speech should follow a response direction rather than create the whole meaning after the fact.

Safe framing:

```text
response direction -> speech
```

Unsafe framing:

```text
speech -> choice
speech -> meaning
```

## Non-Goal

The first public artifact is not meant to prove a large claim.

It is not meant to prove:

- consciousness;
- life;
- mature autonomy;
- live behavior;
- actual selected results.

It is meant to show one small inspectable achievement:

> body-contour distinction before speech, with the boundary preserved.

## Final Reminder

The first useful artifact should open a calm review surface.

Show the achievement.

Keep the boundary visible.

Do not expose the protected mechanism.

Do not make the artifact sound smaller than it is.

Do not make the artifact sound larger than the evidence can hold.

## One-Line Summary

The first public artifact should be small, bounded, and reviewable: show body-contour distinction before LLM speech without implying live behavior, memory authority, actual selected results, runtime proof, consciousness, life, or a finished system.
