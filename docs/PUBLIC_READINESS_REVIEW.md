# Public Surface Review

This is a public-surface review note for Garmon World.

It describes what the repository currently shows, what it may safely say, what it must not imply, and how public updates should be reviewed.

The repository is no longer only a quiet doorway with future plans.

It now has a clear public-safe achievement: Body Shadow and body-contour distinction before speech, plus a bounded static chain up to a synthetic pre-selected result boundary candidate.

The goal of this review is simple:

> show the achievement clearly, keep the boundary visible, and do not expose the private mechanism.

Last reviewed: 2026-05-12.

## Public Surface Role

Garmon World is a public research surface.

It should help a careful reader understand Garmon’s central question:

> can an artificial answer have an origin before speech?

The public surface should explain the idea, the current supported material, the working theory, and the open hypotheses.

It should not turn the repository into:

- a private implementation dump;
- a product landing page;
- a claim of public live behavior;
- a proof of consciousness, life, or mature autonomy;
- a normal chatbot or agent-wrapper description;
- a public runtime manual.

The public surface should be strong enough to show that Garmon is not empty.

It should be bounded enough to show that Garmon is not overclaimed.

## Main Public Position

The public surface may state Garmon’s positive claim directly:

> Garmon studies the order in which an artificial answer may receive an origin before speech.

A compact public chain may be used:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This chain is not a claim that the full private working contour is publicly open.

It is the public research frame for understanding why Garmon does not reduce behavior to final text alone.

## Broader Working Direction

The repository may gently explain the broader working direction behind Garmon.

This direction can be summarized as:

```text
internal state -> meaning -> choice boundary -> time/trace -> speech
```

This is not a public claim of live choice.

It is a careful way to explain why Garmon does not reduce behavior to final text.

The public surface may say that:

- convincing speech is not the same as a stable line of behavior;
- internal state can shape how an event is read;
- meaning should become visible before final wording;
- memory or residue should be framed as trace, not as hidden motor;
- speech should be framed as an expression layer, not as the source of choice;
- evidence helps avoid self-deception, but evidence is not truth, choice, permission, consciousness, life, or autonomy;
- visibility, influence, authority, and choice should remain separate.

This broader direction should support the public understanding of Garmon.

It should not replace the project with an abstract theory manifesto detached from Garmon.

## Sharing Guidance

Recommended behavior when sharing the repository:

- lead with the achievement, not only boundaries;
- keep the boundary short and visible;
- do not announce everywhere at once;
- check that README renders correctly;
- check that links work without login, if the repository is public;
- keep Issues calm and narrow;
- save useful feedback;
- do not answer sharp comments immediately;
- do not add stronger claims in response to attention;
- do not publish private details to explain the project faster;
- do not make replies stronger than the repository wording;
- do not frame Garmon as a finished product, conscious system, living system, or mature autonomous system;
- do not let the broader theory overpower the concrete public achievement.

Public sharing should be slow enough to preserve accuracy.

It should not be so cautious that the real achievement disappears.

## Stop Conditions

Pause public sharing if:

- any safety check fails;
- Vitaliy feels rushed;
- README feels unclear;
- README sounds too defensive;
- README sounds too strong;
- the opening post sounds like a product launch;
- Body Shadow is framed as live behavior;
- static/shadow evidence is framed as implementation permission;
- the pre-selected result boundary candidate is framed as an actual selected result;
- private material appears in the repository;
- memory or residue sounds like active live authority;
- speech sounds like the source of choice;
- reports sound like decision truth;
- evidence sounds like permission;
- the broader theory language starts to overpower the concrete public achievement;
- Garmon sounds like a proven conscious or living system;
- Garmon sounds like an empty disclaimer project;
- the first public link would be sent to a hostile or noisy audience.

Stop conditions are not failures.

They are signals to restore the boundary before moving wider.

## Review Before Each Public Update

Before updating public material, check:

- Does the update make the public surface clearer?
- Does it show the achievement without hype?
- Does it expose any private mechanism?
- Does it make a claim stronger than the evidence supports?
- Does it confuse supported evidence with theory?
- Does it confuse theory with open hypothesis?
- Does it confuse static/shadow evidence with live behavior?
- Does it confuse visibility with authority?
- Does it confuse memory or residue with chooser authority?
- Does it confuse speech with choice?
- Does it confuse report with decision truth?
- Does it keep Garmon as the main project, not an abstract theory detached from it?
- Does it preserve answer origin before speech as the central idea?
- Does it sound too apologetic?
- Does it sound too sensational?
- Does it help a careful outsider understand what can be reviewed?

## Public Artifact Review

Before publishing a public artifact, check:

- Is the artifact small enough to review?
- Does it show body-contour distinction clearly?
- Does it avoid pretending to be a full demo?
- Does it separate input, contour, meaning candidate, and boundary?
- Does it avoid private implementation details?
- Does it include negative controls if useful?
- Does it show what is supported?
- Does it say what is not supported?
- Does it avoid implying live choice?
- Does it avoid implying memory authority?
- Does it avoid implying speech authority?
- Does it make the achievement easier to inspect?
- Does it keep the private mechanism private?

## Public Response Review

When answering public questions or comments:

- answer calmly;
- do not overexplain private internals;
- do not strengthen claims under pressure;
- do not publish private examples to win an argument;
- do not answer hostile comments immediately;
- separate supported material, theory, and hypothesis;
- repeat boundaries only where they clarify, not as panic;
- keep the central idea visible;
- invite precise review rather than broad validation;
- correct misunderstandings without sounding defensive.

A good public reply should leave the reader clearer, not more impressed by noise.
