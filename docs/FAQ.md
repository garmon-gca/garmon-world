# FAQ

## What is Garmon?

Garmon is an early research prototype studying answer origin before speech.

The central public idea is simple:

> an artificial answer may have a path before it becomes final words.

In Garmon, the LLM writes the final text, but the project studies what may happen before those words are written:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This public frame does not claim consciousness, biological life, subjective experience, or mature autonomy.

It is a research direction for making the pre-speech layer easier to inspect.

## What is body-born meaning?

Body-born meaning is a public term used in Garmon World.

It means that an event may receive meaning through an internal-state layer before final LLM speech.

This does not mean:

- biological simulation;
- human feeling;
- consciousness;
- life;
- a finished autonomous system.

It means that the event is not treated as plain text only.

The event first meets an internal background, and that background may shape the meaning direction before final wording.

## What is the broader Garmon direction?

Garmon studies the path before final speech.

A compact public frame is:

```text
event -> public event reading -> response direction -> LLM speech -> final answer
```

A broader working direction may be described as:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This is not a public claim of live choice or live behavior.

It means that Garmon tries to keep state, meaning, memory, choice, behavior, truth, evidence, and speech from collapsing into one text-generation step.

## What has been shown so far?

The clearest current public-safe achievement is Body Shadow.

Static/shadow checks show that several body-contour candidates remain distinguishable before LLM speech:

- quiet baseline;
- fatigue;
- recovery;
- stress;
- mixed stress/recovery;
- day/night rhythm.

These contours do not collapse into one generic label.

The public materials also describe a closed static choice boundary chain:

```text
body contour
-> meaning candidate
-> passive context
-> choice visibility
-> choice ownership
-> behavior boundary
```

A later public-safe update extends the chain to a synthetic pre-selected result boundary candidate.

This is a marker near the door before selected result.

The door remains closed.

There is no public claim of:

- actual selected result;
- live choice;
- public behavior;
- memory influence on choice;
- application authority;
- runtime proof.

## Why does Body Shadow matter?

Body Shadow matters because it makes part of the pre-speech layer easier to inspect.

A final answer alone may hide whether different internal-state contours were preserved or collapsed into one generic response shape.

Body Shadow gives Garmon a public-safe review surface where contour differences remain visible before final wording.

The achievement is not stronger wording.

The achievement is inspectability before speech.

## Is Body Shadow a command system?

No.

Body Shadow is a review surface for body-contour distinction.

A body contour may be inspected before speech, but it does not become:

- command authority;
- memory;
- behavior;
- truth;
- final speech;
- selected-result authority;
- implementation permission.

The central law is:

```text
participation is not power
```

A contour may participate in the path without becoming the owner of the answer.

## Is Garmon a chatbot?

Not exactly.

A chatbot usually centers on conversation and final replies.

Garmon focuses on what happens before speech appears.

The repository is currently a public research preview, not an interactive public chat system.

## Is Garmon a new language model?

No.

Garmon does not train a new base model.

It studies the path around the LLM speech layer:

- event reading;
- body-contour distinction;
- meaning direction;
- boundaries;
- trace;
- reviewable public examples.

The LLM remains important.

It writes the final words.

But it is not presented as the whole system or as the source of choice.

## Is speech unimportant in Garmon?

No.

Speech is very important.

Speech is how the internal path becomes visible to a human reader.

But Garmon does not treat final speech as the whole system.

In Garmon’s public framing:

```text
response direction -> speech
```

not:

```text
speech -> choice
speech -> meaning
```

Speech expresses the path.

Speech should not replace the path.

## What does “memory as trace” mean?

It means that memory-like material should be treated as trace rather than hidden motor.

A past event may leave:

- caution;
- fatigue;
- recovery pressure;
- changed sensitivity;
- another bounded after-effect.

But the past must not choose instead of the present.

Safe public framing:

```text
past event -> trace -> changed sensitivity -> present meaning
```

Unsafe framing:

```text
past event -> hidden command -> current result
```

The repository does not currently claim:

- active body memory;
- memory influence on live choice;
- lived biography;
- residue-driven behavior.

Memory and residue remain future-gated evidence.

## What is a pre-selected result boundary candidate?

It is a synthetic marker just before an actual selected result.

It means the boundary near selected result has been named for review.

It does not mean the boundary has been opened.

Correct public meaning:

```text
boundary named
boundary not opened
```

The repository does not claim public live choice or actual selected result authority.

## What does “participation is not power” mean?

This is one of the core public laws of Garmon.

It means that a layer may participate in answer formation without becoming the owner of the answer.

Examples:

- body contour may participate without becoming command;
- meaning may participate without becoming selected result;
- memory/trace may participate without becoming hidden motor;
- speech may participate without becoming source of choice;
- evidence may participate without becoming permission.

This law keeps the boundaries clear.

## Why is there no interactive demo yet?

Because the first honest public step does not need to be a live chat system.

A smaller artifact may be more useful first.

For example:

- one event;
- several body contours;
- several public-safe readings;
- visible distinction before speech;
- visible boundary notes.

This is easier to review honestly than a large unclear demo.

## What is the first public artifact supposed to show?

The first artifact should show one narrow result:

> body-contour candidates remain distinguishable before speech.

It should remain:

- small;
- bounded;
- reviewable;
- public-safe.

It should not imply:

- live behavior;
- actual selected result;
- memory authority;
- runtime proof;
- consciousness;
- life;
