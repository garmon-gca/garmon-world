# Review Questions

These are the first questions for technical readers.

## Behavior

- How can we test whether context from previous steps improves behavior rather than only changing the wording?
- What minimal checks are useful for comparing behavior with and without previous context?
- Which parts of the runtime path should be visible in a public demo?

## Reproducibility

- What is the smallest demo that is still meaningful?
- How can a demo avoid private internals while still showing enough evidence?
- What should be logged in a public-safe way?

## Safety

- How can behavior boundaries be shown without exposing private control details?
- How can the project avoid strong claims before external review?
- What wording keeps the work honest and technically useful?

## Agent Design

- Where should speech end and behavior preparation begin?
- How can an LLM-based agent avoid becoming only a prompt bundle?
- What evaluation would distinguish useful runtime context from text noise?
