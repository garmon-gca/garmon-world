# Project Overview

Garmon is a public research preview for studying answer origin before speech.

The central idea is simple:

> an artificial answer may have a path before it becomes final words.

Garmon does not treat the LLM as the whole system. The LLM is used as the speech layer: the part that turns an already shaped direction into human-readable language.

The path before speech is studied separately.

A request is not only a text task. It can be treated as an event that meets an internal state, receives a body contour, becomes meaningful, approaches a choice boundary, leaves a trace, and only then reaches speech.

A compact public frame is:

```text
state -> body contour -> meaning -> choice boundary -> trace -> speech
```

This repository explains that public frame, the current public-safe achievement, and the boundaries that keep the claim accurate.

## Core Idea

Many AI systems are viewed through this surface pattern:

```text
request -> analysis -> plan -> answer
```

That pattern can be useful, but it leaves the deepest question open:

> where did the answer come from?

Garmon explores another question:

> what happens if the final words are written by an LLM, but the event receives meaning before those words are written?

This is the shift:

```text
event -> internal state -> body contour -> meaning direction -> speech
```

The goal is not to make the final answer sound more dramatic.

The goal is to make the pre-speech path easier to inspect without treating final text as the whole system.

## Body-Born Meaning

In these public notes, **body-born meaning** means a high-level event-reading step before final LLM speech.

It means that an event is not treated as plain text only.

The event first meets an internal-state layer. That meeting can give the event a contour: quiet baseline, fatigue, recovery, stress, mixed stress/recovery, rhythm, pressure, caution, or another public-safe form of internal weight.

That contour may support a meaning candidate before final wording.

This is not a biological claim.

It is not a claim of subjective experience.

It is not a claim that Garmon copies a human body.

It is a public way to describe a pre-speech meaning layer.

## The Role of the LLM

The LLM remains important.

It writes the final words.

It can clarify, translate, soften, explain, and shape the answer into language.

But in Garmon, speech is not treated as the source of the whole system.

Speech should express a path.

Speech should not replace the path.

A good public description is:

```text
LLM = speech layer
```

This does not weaken the LLM.

It gives the LLM a precise role.

## Working Direction Behind Garmon

Garmon grew from a practical problem:

> convincing speech is not the same as a stable line of behavior.

A system can answer clearly and intelligently while still reducing behavior to final text or restarting its behavioral line from session to session.

A prompt can restore style, role, or instruction.

But a prompt alone does not create continuing internal state.

Garmon explores a different framing:

- speech should express a path, not replace it;
- internal state should shape how an event is read;
- body contour should give the event internal weight;
- meaning should appear before final wording;
- memory or residue should remain trace, not hidden motor;
- choice-related boundaries should be protected from speech, memory, reports, and external commands;
- evidence should help avoid self-deception, but evidence is not truth, choice, permission, or autonomy.

The short public law is:

```text
participation is not power
```

A layer may participate in answer formation without becoming the owner of the answer.

## What Has Become Visible

Recent public-safe work makes part of the pre-speech path more concrete.

Static/shadow Body Shadow checks show that several body-contour candidates remain distinguishable before LLM speech.

Public-safe examples include:

- quiet baseline;
- fatigue;
- recovery;
- stress;
- mixed stress/recovery;
- day/night rhythm.

These contours do not collapse into one generic label.

This gives Garmon a more inspectable pre-speech review surface.

The achievement is inspectability before speech.

It is not louder wording.

It is not a claim that the private mechanism is published.

It is not a claim of public live behavior.

## Current Public-Safe Chain

A static/shadow evidence chain also preserves boundaries across the pre-speech path:

```text
body contour
-> meaning candidate
-> passive context
-> choice visibility
-> choice ownership
-> behavior boundary
-> pre-selected result boundary candidate (synthetic)
```

This chain means that body-contour candidates can move toward meaning and passive context while preserving the boundary that they do not become:

- command authority;
- memory;
- behavior;
- truth;
- speech authority;
- selected-result authority;
- runtime proof;
- implementation permission.

The final link, the **pre-selected result boundary candidate**, is a synthetic marker.

It identifies the door just before an actual selected result.

The door remains closed.

Correct public meaning:

```text
boundary named
boundary not opened
```

There is no public claim of an actual selected result, live choice, public behavior, active memory authority, application authority, or runtime proof.

## Public Stage

This repository explains:

## Why This May Matter

Garmon may matter because many questions in AI behavior are hidden when we inspect only the final answer.

A final answer can be useful and still hide whether a system preserved meaningful pre-speech differences or collapsed them into one generic response shape.

Possible future relevance includes:

- safer response shaping;
- believable digital characters;
- long-running assistants;
- game or NPC systems;
- robotics interfaces;
- technical review of meaning before speech;
- distinguishing useful prior context from text noise;
- clearer separation between state, meaning, memory, choice, behavior, truth, and speech.

The more specific current value is narrower:

> body-contour distinction before final text can be reviewed publicly.

That is already useful.

It gives reviewers something more concrete than the final answer alone.

## Current Public Milestone

Garmon now has a public-safe static/shadow evidence layer.

It shows:

- distinguishable body-contour candidates before speech;
- negative controls that catch forbidden authority leaks;
- static sequence coherence between simple contour transitions;
- preserved boundaries around command, memory, speech, behavior, truth, selected result, runtime proof, and implementation permission;
- a closed static choice boundary chain from body contour through behavior boundary;
- a passive chain extended to a synthetic pre-selected result boundary candidate.

The next useful public step is a small reviewable artifact that shows this contour distinction and boundary clearly without exposing private internals.

## Boundaries of This Overview

This overview does not claim that:

- Garmon is a finished product;
- Garmon is a new trained language model;
- Garmon is a general intelligence;
- Garmon is conscious;
- Garmon is alive;
- Garmon has proven subjectivity;
- Garmon has mature autonomy;
- Garmon has public live behavior;
- Garmon has public actual selected results;
- memory or residue currently drives live choice;
- speech is the source of choice;
- static/shadow evidence proves the whole theory;
- public summaries reveal the private mechanism.

These boundaries do not make the project empty.

They keep the public claim accurate.

## One-Line Summary

Garmon is a public research preview for studying a bounded pre-speech path: internal state can shape meaning before final LLM speech, while memory, choice, behavior, truth, evidence, and speech remain separated and reviewable.
