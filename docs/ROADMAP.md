# Roadmap

This roadmap is intentionally small.

## Step 1: Public Entry Point

- Create a clean repository.
- Explain the project in careful public language.
- Publish boundaries and review questions.

## Step 2: Public-Safe Demo Plan

- Design a small demo.
- Keep private internals out of the demo.
- Define what can be logged safely.
- Define what the demo does and does not show.

## Step 3: Reproducible Demo

- Add a minimal runnable demo when it is ready.
- Make the setup understandable.
- Include expected outputs and failure cases.
- Keep the claims modest.

## Step 4: External Review

- Invite calm technical feedback.
- Improve the demo based on review.
- Keep unclear results marked as unclear.

## Not On The Roadmap Yet

- product launch;
- public API;
- commercial deployment;
- large community promises;
- strong claims about the nature of the agent.
